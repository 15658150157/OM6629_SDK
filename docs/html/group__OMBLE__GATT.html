◆ ob_gattc_write()

uint32_t ob_gattc_write ( uint8_t  conn_idx,
uint16_t  att_hdl,
uint8_t  type,
const uint8_t *  data,
int  len 
)

Write request.

As a GATT Client, write data to att_hdl. After peer responds, it will notify the upper layer through the event OB_GATTC_EVT_WRITE_RSP

If the parameter type type is OB_GATTC_WRITE_CMD, no response event OB_GATTC_EVT_WRITE_RSP

If the parameter type type is OB_GATTC_WRITE_CMD, after the data is sent to the peer device, the upper layer will be notified through the OB_GATT_EVT_TX_COMPLETE event. You can call ob_gattc_write continuously to cache the data to be written to increase the communication rate

Parameters
[in]conn_idxconnection index
[in]att_hdlatt handle
[in]typewrite type, refer to ob_gattc_write_type
[in]datadata
[in]lendata length, automatically truncated if it exceeds the current MTU-3
Note
When calling ob_gattc_write to send data continuously, please note that the unsent data needs to occupy memory for buffering, so the amount of buffered data should be controlled according to the OB_GATT_EVT_TX_COMPLETE event to prevent memory exhaustion
Returns
result, refer to ob_error

◆ ob_gatts_read_response()

uint32_t ob_gatts_read_response ( uint8_t  conn_idx,
uint16_t  att_state,
const uint8_t *  data,
uint16_t  len 
)

Response to read request.

Used to respond to the attribute requested by OB_GATTS_EVT_READ_REQ. The requested attribute handle value refers to ob_gatts_evt_read_req_t

Parameters
[in]conn_idxconnection index
[in]att_stateRead status, refer to ob_gatt_error_t. If it is not success, the data parameter is useless
[in]dataData
[in]lenLength. If it exceeds the transmittable length, the data content will be automatically truncated
Note
The transmittable length of the data is the current MTU-1. If the data length exceeds, it will be automatically truncated
If the attribute data content read exceeds MTU-1, the GATT Client will initiate a read operation again, and the offset in the read parameter ob_gatts_evt_read_req_t is the offset of the data to be read
If the read operation needs to return an error, set att_state to the response error code. For GATT error codes, refer to ob_gatt_error_t
Returns
result, refer to ob_error

◆ ob_gatts_send_hvx()

uint32_t ob_gatts_send_hvx ( uint8_t  conn_idx,
const ob_gatts_hvx_t *  hvx 
)

Send notification or indication, can only be called when acting as a GATT server.

After sending notification, when the data is sent, you will receive the OB_GATT_EVT_TX_COMPLETE event, but before receiving this event, you can call ob_gatts_send_hvx continuously to cache the notifications to be sent to increase the communication rate

After sending the indication, the peer device will respond to the OB_GATTS_EVT_INDICATE_CFM event

Parameters
[in]conn_idxconnection index
[in]hvxdata parameter
Note
When calling ob_gatts_send_hvx to send notifications continuously, please note that the unsent data needs to occupy memory for caching, so the amount of cached data should be controlled according to the OB_GATT_EVT_TX_COMPLETE event to prevent memory exhaustion
Returns
result, refer to ob_error
