◆ ob_gattc_write()

uint32_t ob_gattc_write ( uint8_t  conn_idx,
uint16_t  att_hdl,
uint8_t  type,
const uint8_t *  data,
int  len 
)

写入请求

作为GATT Client向att_hdl写入数据，对方响应后会通过事件OB_GATTC_EVT_WRITE_RSP 通知上层

若参数type类型为OB_GATTC_WRITE_CMD，则不会产生响应事件OB_GATTC_EVT_WRITE_RSP

若参数type类型为OB_GATTC_WRITE_CMD，数据发送给对端设备后，会通过OB_GATT_EVT_TX_COMPLETE 事件通知上层，可以连续调用ob_gattc_write来缓存需要写入的数据以提高通讯速率

Parameters
[in]conn_idx连接index
[in]att_hdlatt handle
[in]typewrite类型，参考ob_gattc_write_type
[in]data数据
[in]len数据长度，超出当前MTU-3则自动截断
Note
调用ob_gattc_write连续发送数据时要注意未发出的数据是需要占用内存进行缓存的，因此应根据OB_GATT_EVT_TX_COMPLETE 事件控制被缓存的数据量以防内存被耗尽
Returns
执行结果，参考ob_error

◆ ob_gatts_read_response()

uint32_t ob_gatts_read_response ( uint8_t  conn_idx,
uint16_t  att_state,
const uint8_t *  data,
uint16_t  len 
)

响应读请求

该接口用于响应OB_GATTS_EVT_READ_REQ 所请求的attribute，请求的attribute handle值参考ob_gatts_evt_read_req_t

Parameters
[in]conn_idx连接index
[in]att_state读状态，参考ob_gatt_error_t ，若不为success，则data参数无用
[in]data数据
[in]len长度，若超出可发送长度则数据内容自动截断
Note
数据的可发送长度为当前MTU-1，数据长度超出时将自动截断
若读取的attribute数据内容超出MTU-1，则GATT Client会再次发起read操作，且读取参数ob_gatts_evt_read_req_t 中的offset为需要读取数据的偏移量
若读取操作需要返回错误，则将att_state设置为响应的错误码即可，GATT的错误码参考ob_gatt_error_t
Returns
执行结果，参考ob_error

◆ ob_gatts_send_hvx()

uint32_t ob_gatts_send_hvx ( uint8_t  conn_idx,
const ob_gatts_hvx_t *  hvx 
)

发送notification或indication，仅作为GATT server时可调用

发送notification后当数据实际发送完成时会收到 OB_GATT_EVT_TX_COMPLETE 事件，但在收到该事件前可以连续调用ob_gatts_send_hvx来缓存需要发送的notification以提高通讯速率

发送indication后对端设备会回应OB_GATTS_EVT_INDICATE_CFM 事件

Parameters
[in]conn_idx连接index
[in]hvx数据参数
Note
调用ob_gatts_send_hvx连续发送notification时要注意未发出的数据是需要占用内存进行缓存的，因此应根据OB_GATT_EVT_TX_COMPLETE 事件控制被缓存的数据量以防内存被耗尽
Returns
执行结果，参考ob_error
