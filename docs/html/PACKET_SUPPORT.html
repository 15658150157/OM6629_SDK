Air Data Rate

The air data rate is the modulated signaling rate the chip uses when transmitting and receiving data. It can be 25kbps to 2Mbps. Using lower air data rate gives better receiver sensitivity than higher air data rate. But, high air data rate gives lower average current consumption and reduced probability of on-air collisions. The air data rate is set by the RF_DR register. A transmitter and a receiver must be programmed with the same air data rate to communicate with each other.

Frequency Setting

Configure frequency points by drv_om24g_set_freq(uint16_t freq, float fract_freq); For example, if you want to configure 2412.378456MHZ frequency points, set freq to 2412 and fract_freq to 0.378456.

DMA

  • DMA realizes automatic data transfer from system RAM to TX FIFO and RX FIFO to system RAM without participation of MCU.
  • As a PTX device, if DMA TX function is enabled, the data in system RAM will be read out by DMA and written into TX FIFO.
  • As a PRX device, when PRX received a packet, the data in RX FIFO will be read out by DMA and written into system RAM automatic.

Address Match

  • Address match Setting om24g_config_t::addr_chk to 1 enables the packet address match.
  • address match has an attribute called data pipe, for a detail description of data pipe see chapter transfer mode section.
  • The packet handler engine will compare the destination address in the packet with the programmed node address in the RX_ADDR_Px(x:0~5) register and the 0x00 broadcast address when om24g_config_t::addr_chk = 0x02 or both 0x00 and 0xFF broadcast addresses when om24g_config_t::addr_chk = 0x03. If the received address matches a valid address, the packet is accepted. If the address match fails, the packet is discarded and receive mode restarted.
  • In different packet frame structure, different address field will be used for address match.
  • If packet structure A is selected, adress0 subfield would be used for address match.
  • If packet structure B is selected, adress1 subfield in head field (or before/after head field) would be used for address match.

RX Termination Timer

  • When chip is a PTX and acknowledgement function is enabled, it has a function for automatic termination of RX after a programmable time. The termination timer starts when in RX state. The timeout is programmable with register FA_SETUP_RETR.ARD. When the timer expires, there’s that different operation.
  • In packet structure A, the radio controller will go to TX MODE and retransmitted last packet until transmit successful or it reaches auto Retransmit Count value.
  • In packet structure B, the radio controller will go to TX MODE.
  • A RX timeout interrupt will happen if the interrupt is enabled.

Bit Count Match

  • The radio implements a simple counter that can be configured to generate an event after a specific number of bits have been received.
  • The bit counter is started by set INT_MASK.MASK_BCM = 1, and stopped by set INT_MASK.MASK_BCM = 0. A INT_BCC_MATCH event will be generated when the bit counter has counted the number of bits specified in the BCC.CNT register. The bit counter will continue to count bits until set INT_MASK.MASK_BCM = 0 or radio is disabled. The MCU can therefore, after a BCMATCH event, reconfigure the BCC.CNT value for new BCMATCH events within the same packet.
  • For packet structure A, the bit counter can only be started after the radio has received the address.
  • For packet structure B, the bit counter can only be started after the radio has received the sync word.
  • The bit counter will stop and reset with INT_MASK.MASK_BCM = 0 or a packet receive is finish.

Timestamp

The software uses the 32bit counter to realize the timestamp function and completes the synchronization process of different device time. If timestamp function is enabled, timestamp field will be transmitted after the payload field. The 4-bytes timestamp is included in payload field. The timestamp of the transmitting device is locked after sending the header. The four byte timestamp is included in the length.

CRC

The CRC polynomial is configurable as illustrated in Figure CRC generation of an n bit CRC where bit0 in the CRCPOLY register corresponds to X0 and bit1 corresponds to X1 etc. Polynomial is set by om24g_config_t::crc_poly. CRC initital value is set by om24g_config_t::crc_init.

CRC generation of mode_0

As illustrated in figure CRC generation of an n bit CRC, the CRC is calculated by feeding the packet serially through the CRC generator. Before the packet is clocked through the CRC generator, the CRC generator's latches b0 through bn will be initialized with a predefined value specified in the CRCINIT register. When the whole packet is clocked through the CRC generator, latches b0 through bn will hold the resulting CRC. This value will be used by the radio during both transmission and reception but it is not available to be read by the MCU at any time. A received CRC can however be read by the MCU via the RXCRC register independent of whether or not it has passed the CRC check.

The length (n) of the CRC is configurable to 1/2/3/4 bytes by om24g_config_t::crc_len to set. If om24g_config_t::crc_len = 0x01, CRC length is one byte and CRC calculation are enabled.

CRC generation of mode_1

After the whole packet including the CRC has been received, the radio will generate a CRCERROR event if CRC errors were detected.

CRC Scope:

In packet structure A, the scope of CRC is fixed: includes sync_word, 1-byte address field, 2 bytes guard field (if guard field exist), header field, and payload field.

In packet structure B, the scope of CRC includes sync_word, header, payload and timestamp field. The scope of CRC for these field is individual optional.

Note
If you need to modify the CRC configuration again, for example, CRC_CFG, CRC_SKIP, you needs to call OM24G_CE_LOW() to turn off the RF, and then perform new data communication (for example, the CRC calculation of ADDR is calculated immediately after the last receive). The CRC_SKIP_ADDR modified later will not work in the current transfer).

Date Whitening

From a radio perspective, the ideal over the air data are random and DC free. This results in the smoothest power distribution over the occupied bandwidth. This also gives the regulation loops in the receiver uniform operation conditions (no data dependencies). Real world data often contain long sequences of zeros and ones. Performance can then be improved by whitening the data before transmitting, and de-whitening the data in the receiver. Selected field data are then XOR-ed with pseudo-random sequence before being transmitted. At the receiver end, the data are XOR-ed with the same pseudo-random sequence. This way, the whitening is reversed, and the original data appear in the receiver. In OM6XXX, this can be done automatically by setting om24g_config_t::white_en = 1.

Seven types of data whitening are supported, which can be selected by setting om24g_config_t::white_sel. In addition, om24g_config_t::white_seed is used to select the initial value of data whitening LSFR, and om24g_config_t::white_obit selects which register output in LSFR is used to perform exclusive OR processing with data. Specific Settings need to be set by users according to their own needs. Schematics corresponding to different whitening types and polynomials are listed below.

Set om24g_config_t::white_sel = 0, then select Onmicro 66xx whitening method: PN8 = x8+x4+x+1. (set om24g_config_t::white_seed = 0x80, om24g_config_t::white_obit = 0x07)

Data Whitening Range:

Data whitening ranges include header field, addr1 field, payload field, and crc field. You can set whitening Fields that are not whitened by setting om24g_config_t::white_skip_addr, om24g_config_t::white_skip_hdr and om24g_config_t::white_skip_crc. When the addr1 field is inside the header, white_skip_addr is invalid. In the case of header whitening, addr1 field is forcibly whitened(FB_PKTCTRL.ADDR1_LOC = 0x00). When the address is outside the header, white_skip_addr is valid.

Demodulation Mode

To prevent improper use by customers, modifying the demodulation mode in the config structure is invalid for 66xx. Forcefully set the demodulation mode for each rate in the drv_om24g_set_rf_parameters() function, which is suitable for most chips' reception, but can be switched in this function if the reception effect is poor. There are three demodulation modes:

SOFT DETECTION

  • The modulation index supports 0.5 and 0.32.
  • The speed supports 1M, 2M, 500K, and 250K, but does not support low-speed mode. Enter low rate mode when EN_TX_ARB or EN_RX_ARB is set to 1.
  • Highest accruacy, highest latency.

HARD DETECTION

  • Suport all modulation index(0~2).
  • The speed supports 1M, 500K, 250K and low-speed mode(25k~250k), but does not support 2M.
  • Middle accruacy, middle latency.

DPLL DETECTION

  • Suport all modulation index(0~2).
  • Supports all rates.(25K~2M).
  • Lowest accruacy, lowest latency.

MSB/LSB Endian

Byte endianness of sync word field、bit endianness of sync word field、byte endianness of CRC field、bit endianness of CRC field、 bit endianness of header field、bit endianness of address field and bit endianness of payload field can be configured through the ENDIAN register.
