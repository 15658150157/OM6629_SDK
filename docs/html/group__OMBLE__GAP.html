◆ ob_adv_properties_bits

enum ob_adv_properties_bits

adv广播属性bit位

若使能了bit位OB_ADV_PROP_BIT4_LEGACY_ADV，则使用传统广播，广播数据长度最大为31字节，广播属性仅支持如下组合:
OB_ADV_PROP_BIT4_LEGACY_ADV | OB_ADV_PROP_BIT1_SCANNABLE | OB_ADV_PROP_BIT0_CONNECTALBE
OB_ADV_PROP_BIT4_LEGACY_ADV | OB_ADV_PROP_BIT2_DIRECTED_ADV | OB_ADV_PROP_BIT0_CONNECTALBE
OB_ADV_PROP_BIT4_LEGACY_ADV | OB_ADV_PROP_BIT3_HIGH_DUTY_CYCLE | OB_ADV_PROP_BIT0_CONNECTALBE
OB_ADV_PROP_BIT4_LEGACY_ADV | OB_ADV_PROP_BIT1_SCANNABLE
若未使能bit位OB_ADV_PROP_BIT4_LEGACY_ADV，则使用扩展广播，广播数据长度最大为251字节

ob_adv_param_t *  adv_param,

ob_data_t *  scan_rsp_data 
)

开启指定adv_idx的广播，或更新广播数据

多广播场景下用参数adv_idx区分不同广播，不同广播支持配置成不同的广播地址
若参数adv_param中的local_addr为NULL时，广播使用ob_gap_addr_set 设置的地址，否则使用local_addr指定的地址

Parameters
[in]adv_idx广播index
[in]adv_param广播参数参考ob_adv_param_t, 若为NULL则只更新当前广播的广播数据
[in]adv_data广播数据
[in]scan_rsp_data扫描数据
Note
若需要更新广播参数，则该广播必须为已开启状态
参数adv_idx值必须小于HOST支持的最大广播数，最大广播数在协议栈初始化时定义，见函数omble_init 的参数ob_stack_param
如果使用的是 ble_1peripheral 的蓝牙库，更新广播数据时，需要先停止广播，再调用此函数以更新广播数据
Returns
执行结果，参考ob_error
